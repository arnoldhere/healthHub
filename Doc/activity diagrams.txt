activity diagrams

1) get service (clientside)
	- start
		login to system (user)
		check/validate user  (system)
			if yes then
				fill details for service (user)
				select service type (user)
				check for service avalibility (system) 
					if yes then 
					allocate the service to staff (admin)
					& send details to user (system)
					else try again
			else try again
	- end 

2) confirm home visit (admin side)
	-start
		login to system (admin)
		check for pending home visit (admin)
			if yes then 
				validate home visit (includes user visit request duration)
					if yes then 
						approve home visit (admin)
						allocate the home visit (admin)
					else try again
			else try again		
	-end
